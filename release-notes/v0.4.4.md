## 0.4.4

### Improved
- Scoped cached `test-status` baselines to the current working directory instead of one global last-run file, so `sift rerun` and `sift escalate` no longer pick up another project's state.
- Clarified the `test-status` cache lifecycle in the CLI reference, including where the baseline lives, when it is written, and how to reset it.
- Reworked the test pyramid so the default suite stays source-backed and fast, with a much thinner smoke layer and separate built-artifact E2E coverage.

### Added
- Added dedicated integration coverage for `exec`, `rerun`, and `escalate` orchestration without relying on a full outer CLI subprocess for every behavior branch.
- Added a separate E2E lane for `dist`, packaging, and docs acceptance flows, plus an E2E-only Vitest global setup.
- Added consistent test-layer suffixes across the remaining legacy `*.test.ts` files (`*.unit.test.ts`, `*.integration.test.ts`, `*.smoke.test.ts`, `*.e2e.test.ts`).

### Changed
- Updated CI and release workflows to use the split test commands explicitly: coverage, smoke, and E2E now run as distinct lanes.
- Updated contributor-facing docs so the README points to `CONTRIBUTING.md`, and `CONTRIBUTING.md` explains the new unit/integration/smoke/E2E boundaries.
- Hardened fake-provider test stability and one-shot provider requests so subprocess-heavy test runs shut down more reliably.
- Bumped package version to `0.4.4`.
