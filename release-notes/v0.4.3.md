## 0.4.3

### Improved
- Reworked the public docs so the README and benchmark notes tell a clearer first-run story: the product value, benchmark caveats, and command guidance are easier to scan without losing detail.
- Hardened setup diagnostics so `sift doctor` now treats obvious placeholder API keys as incomplete configuration instead of reporting a false healthy state.
- Hardened config template writes so `sift config init` uses the same secure file-permission posture as the guided config writer.

### Added
- Added a lightweight `CONTRIBUTING.md` for first-time contributors.
- Added simple GitHub issue forms for bug reports and feature requests, plus a security contact link for private vulnerability reporting.
- Added regression tests covering placeholder API-key detection in `doctor` and secure template-config writes.

### Changed
- Updated the security policy to reflect the current supported version line and point reporters to GitHub's private advisory flow.
- Bumped package version to `0.4.3`.
