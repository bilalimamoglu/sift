## 0.3.2

### Improved
- Expanded `test-status` with support for 18 more specific pytest failure families, including timeouts, async event loop failures, permission and port-binding failures, fixture teardown failures, database migration failures, configuration errors, xdist worker crashes, django DB access failures, network failures, segfaults, serialization failures, missing file failures, memory errors, deprecation-as-error warnings, strict xfail passes, resource leak warnings, and flaky rerun detection.
- Improved `test-status` diagnosis output with bucket-specific labels, confidence, anchors, search hints, and fix guidance so common failures are more actionable without reading raw traceback first.
- Improved the manual release workflow so it prefers versioned release notes files when present and falls back to GitHub-generated notes when they are not.

### Added
- Added support for versioned release notes files in the manual release workflow:
  - `release-notes/v<version>.md`
  - `release-notes/<version>.md`

### Changed
- Bumped package version to `0.3.2`.
