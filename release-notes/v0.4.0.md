## 0.4.0

### Improved
- Improved `test-status` with richer, more actionable diagnosis output:
  - per-bucket `suspect_kind` and `fix_hint`
  - top-level `primary_suspect_kind` and `confidence_reason`
  - cleaner standard renders with a `Likely owner` line only when it adds signal
- Improved `test-status` ergonomics for real debugging loops:
  - trivial pass-only and short obvious outputs stay compact
  - provider follow-up failures no longer leak raw parse errors into user-facing text
  - exact literal and golden-output drift cases are bucketed more precisely
  - assertion/runtime ownership is inferred more conservatively from real file-path evidence
- Improved `rerun --remaining` from a pytest-only remaining workflow into runner-aware remaining analysis:
  - `pytest` keeps narrowed subset reruns
  - `vitest` and `jest` can now rerun the original full command and report only what still remains relative to the cached baseline

### Added
- Added benchmark-backed `test-status` showcase coverage with frozen companion outputs and example pages for:
  - mixed pytest suites
  - mixed vitest failures
  - public summary-first diagnose JSON
- Added a historical incident archive for the Phase 4a debugging cycle so the before/after behavior that shaped recent fixes is preserved in-repo.
- Added benchmark history tooling and reports so version-over-version reduction trends can be regenerated locally.
- Added `remaining_mode` to public diagnose JSON so automation can distinguish full runs, pytest subset reruns, and full-rerun remaining diffs.

### Changed
- Updated README onboarding to make quick try, config setup, doctor, and agent installation more explicit for new users.
- Updated CLI, docs, and managed agent guidance to reflect the new `rerun --remaining` semantics for `pytest`, `vitest`, and `jest`.
- Bumped package version to `0.4.0`.
