# Benchmark History

These files are generated by:

- `npm run bench:report`
- `npm run bench:history`

- `current-benchmark-report.json` and `current-benchmark-report.md` describe the latest local benchmark state.
- `version-history.json` and `version-history.md` compare tagged releases from a chosen starting tag.
- `snapshots/` is optional and can be populated with versioned snapshots via:
  - `npm run bench:report -- --snapshot v0.4.0`
  - or by checking in the generated `version-history.*` files after a release comparison refresh

Recommended usage:

1. Run `npm run bench:report` during active development when you want the current benchmark picture.
2. Run `npm run bench:report -- --snapshot <label>` before a release or major heuristic change.
3. Run `npm run bench:history` when you want to see how `test-status` changed across tagged versions since `v0.3.0`.
4. Commit the snapshot or history report if you want explicit trend comparisons in git.

This keeps the current report easy to refresh while still allowing explicit version-by-version tracking.
